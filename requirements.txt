# PR31: torch (a sentence-transformers dependency) defaults to the CUDA
# build on PyPI, which pulls several GB of unused nvidia-*/cuda-toolkit
# packages in a GPU-less environment like Codespaces. This index is
# listed first so pip prefers its much smaller CPU-only wheels instead.
# Discovered the hard way: a plain `pip install sentence-transformers`
# installed ~2GB of CUDA libraries in a Codespace with no GPU at all.

# pick the default PyPI CUDA build even with that extra index present.

# Local LLM classification fallback for rounding need classification
# (PR34/B) -- the third and most expensive tier of the classification
# chain, see backend/services/llm_classification_service.py. Unlike
# sentence-transformers, llama-cpp-python has NO torch dependency and
# stays lightweight on disk -- but unlike every other ML dependency in
# this file, it has no prebuilt wheel on PyPI and compiles its C++
# backend (llama.cpp itself) from source on every `pip install`. This
# can take several minutes even on a reasonably fast machine; it is not
# stuck, just compiling. The GGUF model weights (~730MB,
# LiquidAI/LFM2.5-1.2B-JP-GGUF) download separately from Hugging Face
# on first actual use, same one-time-download pattern as PR29's
# faster-whisper and PR31's sentence-transformers.
