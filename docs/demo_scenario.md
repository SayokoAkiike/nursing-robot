# デモシナリオ（1分デモ台本）

## 登場人物

| 役割 | 担当 |
|------|------|
| 患者役 | タブレット（患者UI） |
| 看護師役 | PC（看護師ダッシュボード） |
| ロボット | 実機 または 状態遷移ダミー＋PyBulletシミュ |

---

## タイムライン

| 時間 | 誰が | 何をする |
|------|------|---------|
| 0:00 | 患者 | 「🚽 トイレに行きたい」ボタンを押す |
| 0:05 | システム | キット選択・ロボット出発 |
| 0:10 | 看護師画面 | リクエスト・リスク・ロボット状態が表示される |
| 0:20 | ロボット | ベッドサイドに到着 |
| 0:25 | ロボット | QRで患者ID照合 |
| 0:35 | ロボット | 照合OK → トレイが上がる |
| 0:40 | 患者画面 | 「立ち上がらずお待ちください」が表示される |
| 0:48 | 看護師 | 確認ボタンを押す |
| 0:52 | システム | キット開放・ログ記録 |
| 1:00 | — | デモ終了 |

---

## 各シーンの詳細

### シーン1：患者リクエスト（0:00〜0:10）

患者UIの「🚽 トイレに行きたい」ボタンを押す。

**患者画面の表示：**
```
✅ リクエストを受け付けました
Request: Toileting assistance
Status: Preparing...
⚠️ 立ち上がらずお待ちください
```

**セリフ：**「患者さんがトイレに行きたいとリクエストを送りました。」

---

### シーン2：ロボット出発（0:10〜0:20）

**看護師画面の表示：**
```
🔔 新しいリクエスト
患者：203号室 / 患者A
リクエスト：トイレ介助
リスク：転倒リスクあり ⚠️
必要キット：トイレ介助キット
ロボット状態：MOVING_TO_BEDSIDE 🤖
```

**セリフ：**「看護師ダッシュボードにリクエスト・リスク・ロボット状態がリアルタイム表示されます。」

---

### シーン3：ID照合（0:20〜0:40）

ベッドサイドのQRコードをカメラで読み取り照合。

**照合結果：**
```
Scanning... 🔍
Patient QR : PATIENT_A_ROOM_203 ✅
Kit QR     : KIT_TOILETING_A    ✅
Match      : OK → トレイを上げます
```

**セリフ：**「QRで患者IDとキットIDを照合します。正しい組み合わせならトレイが上がります。」

---

### シーン4：待機メッセージ（0:40〜0:50）

**患者画面（更新）：**
```
🤖 ロボットが到着しました

⚠️ 立ち上がらずお待ちください
看護師がまもなく参ります

Please do not stand up.
A nurse will arrive shortly.
```

**セリフ：**「患者さんの画面に待機メッセージが出て、一人立ち上がりを防ぎます。」

---

### シーン5：キット開放（0:50〜1:00）

看護師が「✅ 確認・キット開放」ボタンを押す。

**ログ記録：**
```
2025-07-01 14:32:01 | Room 203 | Patient A | Kit: Toileting | COMPLETED ✅
```

**セリフ：**「看護師の確認後にキットが開放されます。ロボットは患者に触れず、医療判断も行いません。看護師が介助に集中できる環境を作ります。」

---

## デモ注意点

- 実機デモの場合：QRマーカーを事前に印刷・設置、緊急停止ボタンを確認
- 画面デモの場合：患者UIと看護師ダッシュボードを2画面並べる
- PyBulletシミュがある場合：シミュ画面を3画面目として並べるとインパクト大
